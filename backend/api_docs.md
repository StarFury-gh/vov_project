# API Документация

## Модуль Awards (Награды)

### Получение всех наград
- **Метод:** GET
- **Путь:** `/awards/`
- **Описание:** Возвращает список всех доступных наград.

### Получение награды по имени
- **Метод:** GET
- **Путь:** `/awards/name/{name}`
- **Описание:** Возвращает информацию о награде по её имени.

### Получение награды по ID
- **Метод:** GET
- **Путь:** `/awards/{award_id}`
- **Описание:** Возвращает информацию о награде по её уникальному идентификатору.

### Добавление новой награды
- **Метод:** POST
- **Путь:** `/awards/`
- **Тело запроса:**
  ```json
  {
    "name": "строка, название награды",
    "description": "строка, описание награды"
  }
  ```
- **Описание:** Создаёт новую награду.

### Удаление награды
- **Метод:** DELETE
- **Путь:** `/awards/`
- **Параметры:** `award_id`
- **Описание:** Удаляет награду по её ID.

### Назначение награды герою
- **Метод:** POST
- **Путь:** `/awards/assign`
- **Тело запроса:**
  ```json
  {
    "hero_id": 1,
    "award_id": 5
  }
  ```
- **Описание:** Назначает существующую награду герою.

### Массовое назначение наград
- **Метод:** POST
- **Путь:** `/awards/m_assign`
- **Тело запроса:**
  ```json
  {
    "hero_id": 1,
    "awards": ["Медаль Победы", "Орден Славы"]
  }
  ```
- **Описание:** Назначает несколько наград одному герою по именам наград.

### Получение наград героя
- **Метод:** GET
- **Путь:** `/awards/hero/{hero_id}`
- **Описание:** Возвращает все награды, полученные героем.

## Модуль Heroes (Герои)

### Получение списка героев
- **Метод:** GET
- **Путь:** `/heroes/`
- **Параметры:**
  - `skip` — количество пропущенных записей (по умолчанию 0)
  - `limit` — лимит записей (1–100, по умолчанию 20)
  - `search` — строка поиска по имени
  - `birth_year_from`, `birth_year_to` — фильтр по году рождения
  - `death_year_from`, `death_year_to` — фильтр по году смерти
  - `award_filter` — фильтр по награде
  - `rank_filter` — фильтр по званию
  - `w_type` — тип героя (по умолчанию "vov")
- **Описание:** Возвращает список героев с возможностью фильтрации и пагинации.

### Получение героя по ID
- **Метод:** GET
- **Путь:** `/heroes/{hero_id}`
- **Описание:** Возвращает полную информацию о герое, включая награды, звание и место захоронения.

### Добавление нового героя
- **Метод:** POST
- **Путь:** `/heroes/`
- **Тело запроса:**
  ```json
  {
    "full_name": "Иван Иванович Иванов",
    "w_type": "vov",
    "birth_date": "1920-01-01",
    "death_date": "1945-05-09",
    "biography": "Полный текст биографии героя"
  }
  ```
- **Описание:** Создаёт нового героя.

### Добавление изображения герою
- **Метод:** POST
- **Путь:** `/heroes/image`
- **Параметры:**
  - `image` — файл изображения
  - `hero_id` — ID героя
- **Описание:** Загружает и привязывает изображение к герою.

## Модуль Admin (Администрирование)

### Проверка прав администратора
- **Метод:** GET
- **Путь:** `/admin/check`
- **Описание:** Проверяет, является ли текущий пользователь администратором.

### Получение администратора по ID
- **Метод:** GET
- **Путь:** `/admin/{id}`
- **Описание:** Возвращает информацию об администраторе по его ID.

### Получение всех администраторов
- **Метод:** GET
- **Путь:** `/admin/`
- **Описание:** Возвращает список всех администраторов.

### Вход администратора
- **Метод:** POST
- **Путь:** `/admin/login`
- **Тело запроса:**
  ```json
  {
    "email": "admin@example.com",
    "password": "secret_password"
  }
  ```
- **Описание:** Аутентифицирует администратора и возвращает токен доступа.

### Создание нового администратора
- **Метод:** POST
- **Путь:** `/admin/create`
- **Тело запроса:**
  ```json
  {
    "email": "newadmin@example.com",
    "password": "new_password"
  }
  ```
- **Описание:** Создаёт нового администратора.

### Удаление администратора
- **Метод:** DELETE
- **Путь:** `/admin/delete/{id}`
- **Описание:** Удаляет администратора по его ID.

## Модуль Requests (Заявки)

### Получение всех заявок
- **Метод:** GET
- **Путь:** `/requests/`
- **Параметры:**
  - `limit` — лимит записей (по умолчанию 100)
  - `offset` — смещение (по умолчанию 0)
- **Описание:** Возвращает список всех заявок на добавление героев.

### Получение заявки по ID
- **Метод:** GET
- **Путь:** `/requests/{request_id}`
- **Описание:** Возвращает полную информацию о заявке по её ID.

### Обновление статуса заявки
- **Метод:** PATCH
- **Путь:** `/requests/{request_id}`
- **Параметры:**
  - `status` — новый статус заявки (например, "approved", "rejected")
- **Описание:** Обновляет статус заявки.

### Удаление заявки
- **Метод:** DELETE
- **Путь:** `/requests/{request_id}`
