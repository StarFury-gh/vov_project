## Краткая документация API

Базовый URL: `/api`

### Аутентификация
Для защищённых эндпоинтов (создание, обновление, удаление) требуется передавать токен в заголовке:
```
Authorization: Bearer <admin_token>
```
Либо можно использовать заголовок `X-Admin-Token: <token>` (упрощённый вариант). Токен задаётся в переменных окружения.

### 1. Герои

#### Получить список героев
- **URL:** `/heroes`
- **Метод:** `GET`
- **Параметры запроса (query):**
  - `skip` (int, опционально) — количество пропускаемых записей (по умолчанию 0)
  - `limit` (int, опционально) — максимальное количество записей (по умолчанию 20)
  - `search` (string, опционально) — поиск по имени (частичное совпадение)
  - `rank_id` (int, опционально) — фильтр по ID звания
  - `birth_year_from` / `birth_year_to` (int, опционально) — фильтр по диапазону годов рождения
  - `death_year_from` / `death_year_to` (int, опционально) — фильтр по диапазону годов смерти
- **Ответ:** JSON-объект с полями:
  - `items` (array) — список героев (каждый элемент содержит `id`, `full_name`, `rank_name`, `photo_url`, `birth_date`, `death_date`, краткая информация)
  - `total` (int) — общее количество записей
  - `skip`, `limit` — переданные параметры
- **Кэширование:** Ответ кэшируется в Redis на 5 минут (ключ зависит от параметров).

#### Получить информацию о герое по ID
- **URL:** `/heroes/{id}`
- **Метод:** `GET`
- **Параметры пути:** `id` (int) — идентификатор героя
- **Ответ:** JSON-объект с полной информацией:
  - `id`, `full_name`, `birth_date`, `death_date`, `rank_id`, `rank_name` (название звания), `biography`, `photo_url`, `awards` (список наград с названиями и датами), `photos` (список дополнительных фото)
- **Кэширование:** Кэшируется на 5 минут.

#### Создать нового героя (требуется аутентификация)
- **URL:** `/heroes`
- **Метод:** `POST`
- **Тело запроса (JSON):**
  ```json
  {
    "full_name": "Иванов Иван Иванович",
    "birth_date": "1920-05-15",
    "death_date": "1945-04-30",
    "rank_id": 1,
    "biography": "Текст биографии...",
    "photo_url": "https://example.com/photo.jpg"
  }
  ```
  (поля `awards` и `photos` добавляются отдельными ручками, либо можно включить в этот запрос — упростим до базовых полей)
- **Ответ:** созданный объект героя (с присвоенным `id`).
- **Примечание:** Инвалидирует кэш списка героев.

#### Обновить героя (требуется аутентификация)
- **URL:** `/heroes/{id}`
- **Метод:** `PUT`
- **Параметры пути:** `id`
- **Тело запроса:** аналогично созданию, но все поля опциональны (обновляются только переданные)
- **Ответ:** обновлённый объект героя.
- **Примечание:** Инвалидирует кэш для данного героя и списка.

#### Удалить героя (требуется аутентификация)
- **URL:** `/heroes/{id}`
- **Метод:** `DELETE`
- **Ответ:** статус 204 No Content.
- **Примечание:** Инвалидирует кэш для данного героя и списка.

### 2. Звания (справочник)

#### Получить список всех званий
- **URL:** `/ranks`
- **Метод:** `GET`
- **Ответ:** массив объектов:
  ```json
  [
    {"id": 1, "name": "Генерал-майор", "sort_order": 10},
    ...
  ]
  ```
- **Кэширование:** Кэшируется на длительный срок (например, 1 час), так как данные редко меняются.

#### (Опционально) Создать/обновить/удалить звание — только для админки
- Аналогичные CRUD-ручки (`POST /ranks`, `PUT /ranks/{id}`, `DELETE /ranks/{id}`) с аутентификацией.

---

### 3. Награды (справочник)

#### Получить список всех наград
- **URL:** `/awards`
- **Метод:** `GET`
- **Ответ:** массив объектов `{id, name, description}`.
- **Кэширование:** Кэшируется.

#### Добавить награду герою (требуется аутентификация)
- **URL:** `/heroes/{hero_id}/awards`
- **Метод:** `POST`
- **Тело:** `{"award_id": 1, "date_awarded": "1944-05-01"}`
- **Ответ:** созданная связь.
- **Примечание:** Инвалидирует кэш героя.

#### Удалить награду у героя (требуется аутентификация)
- **URL:** `/heroes/{hero_id}/awards/{award_id}`
- **Метод:** `DELETE`
- **Ответ:** 204.

---

### 4. Фотографии

#### Добавить дополнительное фото герою (требуется аутентификация)
- **URL:** `/heroes/{hero_id}/photos`
- **Метод:** `POST`
- **Тело:** `{"url": "...", "caption": "...", "sort_order": 1}`
- **Ответ:** созданный объект фото.

#### Удалить фото (требуется аутентификация)
- **URL:** `/photos/{photo_id}`
- **Метод:** `DELETE`

---

### 5. Аутентификация (если используется JWT)

#### Вход администратора
- **URL:** `/auth/login`
- **Метод:** `POST`
- **Тело:** `{"username": "...", "password": "..."}`
- **Ответ:** `{"access_token": "...", "token_type": "bearer"}`

#### Проверка токена
- **URL:** `/auth/verify`
- **Метод:** `GET`
- **Заголовок:** Authorization
- **Ответ:** информация о пользователе или 401.

---

### Примечания
- Все даты передаются в формате `YYYY-MM-DD`.
- При ошибках API возвращает стандартные HTTP-статусы (400, 401, 404, 500) с JSON-телом `{"detail": "сообщение"}`.
- Для пагинации используются параметры `skip` и `limit` (аналог offset/limit).
